# Fix Workflows and Scripts

## Problem Summary
Multiple workflow YAML files and build scripts reference `aglint` without the required `@adguard/` npm scope, causing AGLint invocations to fail. Additionally, `Scripts/build.sh` has regex compatibility bugs when `rg` (ripgrep) is not installed, and uses a broken runner pattern for invoking local binaries. `Scripts/lib-common.sh` returns `bunx --bun` but the standalone `bunx` binary may be unavailable.

### 4. `Scripts/lib-common.sh`
- `jsrun()` currently returns `bunx --bun` but the `bunx` standalone binary may be absent; change to `bun x` for broader compatibility:
  ```bash
  jsrun() { has bun && echo "bun x" || has npx && echo "npx -y" || echo ""; }
  ```

### 5. `Scripts/build.sh`
- Add `run_runner()` helper that invokes the runner and its flags as proper argv:
  ```bash
  run_runner(){
    if has bun; then
      bun x "$@"
    elif has npx; then
      npx -y "$@"
    else
      "$@"
    fi
  }
  ```
- Fix `rg()` to return `grep -E` when `rg` is unavailable:
  ```bash
  rg(){ [[ -n $_RG ]] && echo "$_RG" || { _RG=$(has rg && echo rg || echo "grep -E"); echo "$_RG"; }; }
  ```
- Fix `lint_filters()`:
  - `npx aglint init` â†’ `run_runner @adguard/aglint init`
  - `npm pkg set scripts.lint=aglint` â†’ `npm pkg set scripts.lint=@adguard/aglint`
- Fix `build_hostlist()`:
  - Replace `chk hostlist-compiler` and direct `hostlist-compiler` calls with `run_runner hostlist-compiler`
  - Return gracefully instead of dying if the compiler is unavailable
- Fix `_process_js()`:
  - Replace `$("$(runner)" esbuild ...)` with `run_runner esbuild ...`
- Export `run_runner` in the parallel-execution block

### 6. `Scripts/update_lists.py`
- Line 380: `["bun", "x", "aglint", ...]` â†’ `["bun", "x", "@adguard/aglint", ...]`

## Rationale
- Scoped packages require the full `@adguard/aglint` name for npm/bun to resolve the executable.
- `grep` without `-E` treats `+`, `|`, and `{}` as literals in BRE, breaking regex patterns in `build.sh` when `rg` is absent.
- `runner()` returns a space-separated string that bash cannot split correctly inside `$()`; `run_runner()` calls the runner directly so bash tokenizes argv properly.
- `bunx` is not guaranteed to be installed as a standalone binary; `bun x` is the universal subcommand available with any Bun installation.
